VIEWS

  • The View in the Structured Query Language is considered as the virtual table, which depends on the result-set of the predefined SQL statement.

  • VIEW hides the complexity of the data and restricts unnecessary access to the database.

  • It permits the users to access only a particular column rather than the whole data of the table.

  • Views also store data in rows and columns, but the rows do not have any physical existence in the database.

  • Any database administrator and user can easily create the View by selecting the columns from one or more database tables.

  • A view can store either all the records of the table or a particular record from the table using the WHERE clause.

CREATE VIEW

        
SYNTAX:

        CREATE VIEW View_Name AS    
        SELECT Column_Name1, Column_Name2, ....., Column_NameN    
        FROM Table_Name    
        WHERE condition;

EXAMPLE:

         mysql> create view duble as select roll_no,marks,age from student;
         Query OK, 0 rows affected (0.02 sec)

         mysql> select *from duble;

         | roll_no | marks | age  |

         |       7 |    20 |   20 |

         +---------+-------+------+

DELETETION OF VIEW

      
SYNTAX:

        DROP VIEW VIEW_NAME;
        
EXAMPLE:

        mysql> drop view duble;
        Query OK, 0 rows affected (0.01 sec)
