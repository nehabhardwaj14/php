COOKIES IN PHP

  • Cookies and sessions are essential for managing state and user data in web applications. This section covers the basics of both.

  • Cookies are small pieces of data stored on the client’s browser that can be used to remember information about the user across sessions.

SETTING TIME IN COOKIE

  • You can set a cookie using the setcookie() function. Here’s how to set a cookie with a time limit:

        <?php
        // Set a cookie that expires in 1 hour
        setcookie("username", "JohnDoe", time() + 3600);
        echo "Cookie 'username' is set!"; // Outputs: Cookie 'username' is set!
        ?>

DELETING COOKIE

  • To delete a cookie, set its expiration time to a time in the past:

        <?php
        // Delete the cookie
        setcookie("username", "", time() - 3600);
        echo "Cookie 'username' has been deleted!"; 
        // Outputs: Cookie 'username' has been deleted!
        ?>

CREATING SESSION COOKIE

  • A session cookie lasts only for the duration of the user's session. Here’s how to create one:

        <?php
        setcookie("session_cookie", "value", 0); // Expires when the browser closes
        echo "Session cookie is set!"; // Outputs: Session cookie is set!
        ?>

INTRODUCTION TO SESSION

  • Sessions allow you to store data across multiple pages. Unlike cookies, session data is stored on the server.

STARTING SESSION

  • To use sessions, you need to start the session at the beginning of your script:

        <?php
        $_SESSION["username"] = "JohnDoe";
        echo "Session variable 'username' is set!"; 
        // Outputs: Session variable 'username' is set!
        ?>

WORKING WITH SESSION VARIABLES

  • To access session variables, simply reference them like this:

      <?php
      echo "Hello, " . $_SESSION["username"]; // Outputs: Hello, JohnDoe
      ?>

DESTROYING SESSION

  • To destroy a session, use the following code:

      <?php
      session_start();
      session_destroy();
      echo "Session destroyed!"; // Outputs: Session destroyed!
      ?>

PASSING SESSION IDs

  • Session IDs can be passed through URLs or forms. Here’s how to retrieve the session ID:

      <?php
      session_start();
      echo "Session ID: " . session_id(); // Outputs: Session ID: 
      ?>

ENCODING/DECODING SESSION

  • You can encode session variables using serialize() and decode them using unserialize():

      <?php
      $_SESSION["data"] = serialize(["item1", "item2"]);
      echo "Data stored in session: " . $_SESSION["data"]; 
      // Outputs: Data stored in session: a:2:{i:0;s:5:"item1";i:1;s:5:"item2";}
      $data = unserialize($_SESSION["data"]);
      echo "First item: " . $data[0]; // Outputs: First item: item1
      ?>

INCREASING SESSION EXPIRE TIME

  • You can increase the expiration time of a session like this:

        <?php
        session_start();
        // Set a new session expire time
        setcookie(session_name(), session_id(), time() + 3600);
        echo "Session expiration time extended!"; 
        // Outputs: Session expiration time extended!
        ?>

WORKING WITH SESSION WITHOUT COOKIES

  • To work with sessions without cookies, you can pass the session ID in the URL:

      <?php
      session_start();
      // Check if session ID is passed in the URL
      if (isset($_GET['PHPSESSID'])) {
          session_id($_GET['PHPSESSID']);
      }
      echo "Session ID used: " . session_id(); // Outputs: Session ID used: 
      ?>
